# SlideCraft Build Animations (路線B — 講者控制節奏)

讓每張 slide 的內容可以逐項出現，按 →（或點擊/tap）才出下一項。像 PPT 的動畫效果，但更輕量。

## 設計原則

1. **不改變 navigation 的基本架構** — → 仍是前進、← 仍是後退。只是在同一張 slide 內，先走完所有 step 才換 slide
2. **data-step 屬性驅動** — 需要逐項出現的元素加上 `data-step="1"`、`data-step="2"`...
3. **CSS transition** — opacity 0→1 + translateY 位移，0.35s ease
4. **所有 step 預設隱藏**，只有 `step-visible` class 才顯示
5. **換 slide 時重設 step 為 0**

### B. 修改每個 layout renderer

每個 layout renderer 需要追蹤一個 `_step` counter，並在該分 step 的元素上加 `data-step` 和 `class="step-item"`。

規則：
- **cover**: title（step 1）、subtitle（step 2）
- **card-list**: 每張 card 整體分 step，不是 card 內的每個 li
- **image-text**: bullets（每個 li 分 step）
- **grid**: 每個 grid box 分 step
- **transition**: ❌ 不分 step（全部一次出現）
- **content**: bullets 分 step；quote 分成 text → source 兩步
- **two-column**: 每個 column heading + items 分 step
- **stat-card**: 每個 stat 分 step
- **timeline**: 每個 step 分 step
- **comparison**: 每個 comparison column 分 step
- **quote**: quote text → cite source 分 2 steps
- **team**: 每個 member card 分 step
- **process-flow**: 每個 process step 分 step

以 `_render_content` 為例（其他 layout 類似）：

```python
def _render_content(content: dict) -> str:
    title = content.get("title", "")
    bullets = content.get("bullets", [])
    quote = content.get("quote")
    _step = 0
    parts = ['<div class="slide-content">']
    parts.append(f'<div class="content-title"><h2>{_esc(title)}</h2></div>')
    parts.append('<div class="content-body">')
    if quote:
        _step += 1
        parts.append(f'<blockquote><p data-step="{_step}" class="step-item">"{_esc(quote.get("text", ""))}"</p>')
        _step += 1
        parts.append(f'<cite data-step="{_step}" class="step-item">— {_esc(quote.get("source", ""))}</cite></blockquote>')
    if bullets:
        parts.append("<ul>")
        for b in bullets:
            _step += 1
            parts.append(f'<li data-step="{_step}" class="step-item">{_esc(b)}</li>')
        parts.append("</ul>")
    parts.append("</div></div>")
    return "".join(parts)
```

### B. 新增 helper 函式

放在 `applyCurrentSizing()` 之前或之後：

```javascript
function getMaxStep() {
  var curSlide = slides[current];
  if (!curSlide) return 0;
  var stepEls = curSlide.querySelectorAll('[data-step]');
  var maxStep = 0;
  for (var i = 0; i < stepEls.length; i++) {
    var s = parseInt(stepEls[i].getAttribute('data-step'), 10);
    if (s > maxStep) maxStep = s;
  }
  return maxStep;
}

function showStep(step) {
  var els = slides[current].querySelectorAll('[data-step="' + step + '"]');
  for (var i = 0; i < els.length; i++) {
    els[i].classList.add('step-visible');
  }
}

function hideStep(step) {
  var els = slides[current].querySelectorAll('[data-step="' + step + '"]');
  for (var i = 0; i < els.length; i++) {
    els[i].classList.remove('step-visible');
  }
}

function resetSteps(slideEl) {
  if (!slideEl) return;
  var els = slideEl.querySelectorAll('.step-item');
  for (var i = 0; i < els.length; i++) {
    els[i].classList.remove('step-visible');
  }
}
```

### C. 修改 `goNext()` 和 `goPrev()`

```javascript
function goNext() {
  var maxStep = getMaxStep();
  if (maxStep > 0 && currentStep < maxStep) {
    // Advance one step within the current slide
    currentStep++;
    showStep(currentStep);
    // Also update counter display if we want
    return;
  }
  // Move to next slide
  currentStep = 0;
  goTo(current + 1);
}

function goPrev() {
  if (currentStep > 0) {
    // Go back one step within the current slide
    hideStep(currentStep);
    currentStep--;
    return;
  }
  // Go to previous slide
  goTo(current - 1);
}
```

### D. 修改 `goTo()`

在 `goTo()` 函式最前面（`var target = ...` 之後，`if (target === current) return;` 之前），加入重置步驟的邏輯。實際上，當 `goTo()` 被 `goNext()` / `goPrev()` 呼叫時（跨 slide 的情況），新 slide 的 step 應該是 0。

在 `goTo()` 裡，當 `current` 確定會改變時，先 reset 舊 slide 的 steps：

```javascript
function goTo(index) {
    var target = Math.max(0, Math.min(index, slides.length - 1));
    if (target === current) return;
    
    // Reset steps on the slide we're leaving
    resetSteps(slides[current]);
    currentStep = 0;
    
    // ... rest of existing goTo logic
}
```

### E. 修改 Init

在 init 區塊（`fitSlides(); updateUI();` 之前），加入清除所有 step 的邏輯：

```javascript
// Reset all step items (hide them)
for (var s = 0; s < slides.length; s++) {
  resetSteps(slides[s]);
}
```
