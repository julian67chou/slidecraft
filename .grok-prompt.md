# SlideCraft Mobile Responsive — Task for Grok Build

Make the HTML slide presentation renderer mobile-responsive. The project is at /workspace/gamma-ppt/

## Current Architecture

1. **renderers/html/renderer.py** — generates self-contained HTML <section> slides with inline CSS
2. **renderers/html/slider.js** — runtime JS for slide navigation (keyboard, touch/swipe, buttons)
3. The slide CANVAS is fixed at 1280x720px (SLIDE_W, SLIDE_H in slider.js)
4. Desktop works by scaling the slide via `transform: scale(var(--slide-scale))` in CSS
5. Mobile currently tries the same approach but scales to ~30% (390px/1280px), making text tiny and content cut off

## What I Need You To Do

### File 1: renderers/html/renderer.py

Inside the function `_style_block()` (around line 72), after all existing CSS (before the final `</style>`), add comprehensive mobile responsive media queries. The `_style_block()` function contains CSS for these layouts: cover, card-list, image-text, grid, transition, content, two-column, stat-card, timeline.

#### A. Base mobile slide (max-width: 800px)
Add this before `</style>`:
```css
@media (max-width: 800px) {
  section.slide {
    width: 100vw !important;
    height: auto !important;
    min-height: 100dvh !important;
    overflow-y: auto !important;
  }
  .slide-content {
    padding: 16px !important;
  }
}
```

#### B. Responsive font sizes for mobile
Add to the same @media block:
- h1: font-size: clamp(22px, 5vw, 36px) !important;
- h2: font-size: clamp(16px, 4vw, 28px) !important;
- h3: font-size: clamp(14px, 3.5vw, 22px) !important;
- .body-text: font-size: clamp(12px, 3vw, 18px) !important;
- .layout-stat-card .stat-grid .stat .stat-value: font-size: clamp(28px, 8vw, 48px) !important;

#### C. Per-layout mobile reflows
Add these inside the same @media block:

1. **layout-card-list**: Grid becomes 1 column
   ```css
   .layout-card-list .card-grid {
     grid-template-columns: 1fr !important;
   }
   ```

2. **layout-two-column**: Columns stack
   ```css
   .layout-two-column .two-col-grid {
     grid-template-columns: 1fr !important;
   }
   ```

3. **layout-image-text**: Row becomes column (image on top, text below)
   ```css
   .layout-image-text {
     flex-direction: column !important;
   }
   .layout-image-text .image-side {
     flex: 0 0 45vh !important;
     min-height: 200px;
   }
   .layout-image-text .text-side {
     flex: 1 !important;
     padding: 16px !important;
   }
   ```

4. **layout-stat-card**: 4 cols → 2 cols
   ```css
   .layout-stat-card .stat-grid {
     grid-template-columns: repeat(2, 1fr) !important;
   }
   ```

5. **layout-grid**: 2x2 → 1 column
   ```css
   .layout-grid .grid-box {
     grid-template-columns: 1fr !important;
     grid-template-rows: auto !important;
   }
   ```

6. **layout-timeline**: Step number circle smaller
   ```css
   .layout-timeline .timeline .step .step-num {
     width: 32px;
   ```

7. **.card-grid** (used by process-flow, team): Stack cards


8. **layout-cover**: Cover title wider


9. **layout-transition h1** and **layout-quote blockquote**: Responsive sizes

10. **body padding** for mobile:

    ```css
    }
    ```

### File 2: renderers/html/slider.js

Modify the `fitSlides()` function:

1. When `isMobile` (viewport < 800px):
   - Do NOT set `--slide-scale` CSS variable to tiny values
   - Just ensure only the current wrapper is visible (display flex, others none)
   - Set `document.body.style.overflow = '';`
   - Do NOT compute or apply any transform scale factors
   - Make wrappers fill viewport naturally

2. Rewrite the mobile branch to be simple:
```javascript
if (isMobile) {
  document.body.style.overflow = '';
  document.body.style.padding = '0';
  for (var j = 0; j < wrappers.length; j++) {
    wrappers[j].style.display = (j === current) ? 'flex' : 'none';
    wrappers[j].style.justifyContent = 'flex-start';
    wrappers[j].style.alignItems = 'stretch';
  }
  nav.style.bottom = '0';
}
```

```css
    transform: none !important;
  }
}
```
